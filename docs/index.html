The why

Autonomous agents are hard.
And expensive.

Running agents that work constantly while you're away — to actually get things done — is a real operations problem. You can't watch them all, you can't remember what each one learned, and the bill is whatever they decided to do without you.

🫥

You can't watch them all

A dozen terminals, a dozen tabs. Which agent is stuck? Which one went rogue? You find out later — usually the expensive way.

🧠

They forget everything

Every session starts from zero. The thing one agent figured out an hour ago is gone, so the next one pays to learn it again.

💸

Coordination costs money

Hand-routing work between agents, re-explaining context, babysitting approvals — that's your time and your tokens, burning while you sleep.

The what

An intelligent multi-agent harness for Claude Code agents that live locally, in your own terminal — wired into a hive mind, given long-term memory, and run by a GOD orchestrator you talk to.

The how

An office you can actually see.

Three things make the whole thing work — and the app makes each one legible on a shared, watchable office floor.

02 / THE MEMORY

MemPalace — a memory layer the whole office shares

Each agent keeps markdown long-term memory, mined into one shared semantic palace. Anything an agent learns is recalled — by any agent — in milliseconds, across sessions.

No more paying to re-learn what the office already knows. You can search the same palace yourself from the UI.

03 / THE ORCHESTRATION

A GOD agent that runs the floor through a hive mind

You talk to one agent. The GOD orchestrator reads every request, routes work to the right specialist, scribes the shared plan, and keeps the system fully autonomous — escalating only truly critical items (spend, destructive ops, scope) into an approvals queue you act on.

Underneath, the hive is a local git repo of plain files: per-agent mailboxes, a blackboard, and an append-only event log. No agent ever touches git.

Get it free

Download, or build from source.

Grab a build for macOS, Windows, or Linux, or run it yourself. You'll need Node 18+, a C/C++ toolchain (for node-pty), and Claude Code on your PATH.

# clone & run
git clone https://github.com/chaitanyagiri/munder-difflin.git
cd munder-difflin
npm install # rebuilds node-pty for Electron
npm run dev # launches the app with hot reload
